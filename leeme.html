Enumeración de las vistas:

Todas las vistas tendrán un header y un pie como fragmentos

  • localhost:8080/: Index principal que se eliminará durante el desarrollo del proyecto.
  • localhost:8080/login: Página con un pequeño formulario para logearse con un usuario de la web.
  • localhost:8080/tienda: Vista sobre la tienda principal, donde se mostrará la lista de productos. Además si eres un administrador, se habilitarán los botones "Administrar Productos" e "Incluir Nuevo Producto".
  • localhost:8080/producto/1: Vista sobre un producto detallado que el usuario ha seleccionado. En la vista, el usuario será capaz de ver todas las ventas, comprar, pujar, vender o fijar un precio.
  • localhost:8080/producto/venta/1: Esta vista, será bastante parecida al de la compra. Detalla un resumen sobre la venta de un producto en el cual el vendedor debe de confirmar 2 checkboxs para verificar que el producto es nuevo y se debe de enviar en 2 días hábiles.
  • localhost:8080/oferta/listaPujas/1: Esta vista mostrará el historial de las pujas que tiene un producto. En el historial se mostrará ibnformación importante del producto, como por ejemplo la talla, el precio y cuántos productos hay disponibles.
  • localhost:8080/oferta/listaPrecios/1: Esta vista mostrará el historial de los precios que tiene un producto. En el historial se mostrará ibnformación importante del producto, como por ejemplo la talla, el precio y cuántos productos hay disponibles.
  • localhost:8080/oferta/listaVentas/1: Esta vista mostrará el historial de las ventas que tiene un producto. En el historial se mostrará ibnformación importante del producto, como por ejemplo la talla, el precio...
  • localhost:8080/admin/formularioProducto: Vista con formulario para poder agregar un producto. Únicamente el administrador será capaz de ver esta vista y poder agregar un producto nuevo.
  • localhost:8080/producto/confirmacionProducto/1: La vista muestra la confirmación de un pedido completado, con un resumen de la factura, detallando el número del pedido y los costes adicionales.
  • localhost:8080/perfil/1: Muestra el perfil del usuario, detallando el historial de pujas pendientes o aceptadas, y productos pendientes de venta o ya confirmados. En el caso de ser administrador, se habilitarán dos botones para ver el listado de usuarios y el listado de productos.
  • localhost:8080/admin/adminUsuarios: Vista para administrar la lista de usuarios. Únicamente el administrador es capaz de ver esta vista. Adicionalmente, cuando entre en el perfil de un usuario, tendrá dos botones para banear o eliminar al usuario.
  • localhost:8080/admin/adminProductos: Vista que se mostrará la lista de los productos. Únicamente el administrador es capaz de ver esta vista y será capaz de añadir, modificar o eliminar los productos.
  • localhost:8080/chat?id=1: (actualmente sin funcionalidad) Consiste en visualizar el chat de un usuario con un asistente virtual de la página.
  • localhost:8080/perfil/depositarFondo/1: En esta vista, el usuario será capaz de añadir nuevos fondos a la página, para seguir pujando o comprando productos de la página.
  • localhost:8080/oferta/fijarPrecio/1: Pagina con un pequeño formulario, donde un usuario que quiera vender un producto puede fijar el precio por el cual quiere vender dicho producto. Se añadirá una oferta (tipo precio) en la BBDD.
  • localhost:8080/perfil/modificarPerfil/1: Página con un pequeño formulario donde el usuario podrá cambiar su contraseña o información de su perfil.
  • localhost:8080/perfil/retirarFondo/1: Página con un pequeño formulario donde el usuario podrá retirar dinero desde su cuenta en nuestra web a su cuenta bancaria. (Operación contraria a la de depositarFondo)
  • localhost:8080/admin/administrarUsuario/1: Pagina donde el administrador podrá visualizar la información relacionada con un usuario en concreto y realizar diferentes acciones (eliminar al usuario, enviarle un mensaje, banearle ...). Aún sin funcionalidad.
  • localhost:8080/admin/modificarProducto/1: Página con un pequeño formulario donde el administrador podrá modificar datos de un producto ya existente (nombre, desc...)
  • localhost:8080/producto/compra/1: Página muy similar a la de venta donde se detallan los datos de una compra y donde el usuario que va a comprar el producto debe confirmar la compra.
  • localhost:8080/oferta/pujar/1: Página con un pequeño formulario donde el usuario podrá pujar por un producto indicando la cantidad de dinero que considere oportuna.
  • localhost:8080/oferta/errorPuja: Página con información sobre los motivos de que la realización de una puja haya fallado.
  • localhost:8080/oferta/eliminarOferta: Página donde se muestra un mensaje indicando el exito de la eliminación de una puja o un precio.

Pruebas:

Solo la prueba de PruebaCompleta.feature se puede ejecutar sin tener que restaurar el servidor.

  • login-Admin.feature:El objetivo de esta prueba es simplemente hacer login en la aplicacion con un usuario ADMIN, mas concretamente con el usuario "TheRealJin".
  • login-danidassler.feature: El objetivo de esta prueba es hacer login en la aplicacion con un usuario de tipo USER, mas concretamente con el usuario "danidassler".
  • login-alvaro09.feature: El objetivo de esta prueba es hacer login en la aplicacion con un usuario de tipo USER, mas concretamente con el usuario "alvaro09".
  • login-sergiom23.feature: El objetivo de esta prueba es hacer login en la aplicacion con un usuario de tipo USER, mas concretamente con el usuario "sergiom23".
  • compra.feature: El objetivo de la prueba es realizar una de las principales actividades de la web, comprar un producto. En este feature hacemos login en la aplicación con el usuario "danidassler" llamando a login-danidassler.feature, después accedemos a la tienda y pinchamos en el primer producto. Una vez en la página del producto, seleccionamos el botón de "comprar ya" y llegaremos a la página de confirmación de compra donde al pulsar el boton de "finalizar compra" habremos realizado la operación al completo. Para comprobar que nuestro saldo se ha reducido y la operación sale en nuestro perfil, accedemos a la pagina del perfil del usuario.
  • puja.feature: El objetivo de la prueba es realizar una de las principales actividades de la web, pujar por un producto. En este feature hacemos login en la aplicación con el usuario "sergiom23" llamando a login-sergiom23.feature, después accedemos a la tienda y pinchamos en el primer producto. Una vez en la página del producto, seleccionamos el botón de "pujar" y llegaremos a la página de puja donde rellenaremos los campos del formulario necesarios para pujar y pulsaremos el boton "pujar", despues de esto, habremos realizado la operación al completo. Para comprobar que nuestro saldo se ha reducido y la operación sale en nuestro perfil, accedemos a la pagina del perfil del usuario.
  • precio.feature: El objetivo de esta prueba es realizar otra de las actividades principales de la pagina web, fijar un precio por un producto que quieres vender. Para ello hacemos login con un usuario tipo USER (alvaro09) y accedemos al primer producto de la tienda (Yeezy 700). Pinchamos en el boton "fijar precio" el cual nos llevara a una pagina con un pequeño formulario donde incluiremos el precio que deseemos. Para comprobar que se ha fijado el precio por nuestro producto nos redirigimos al perfil donde debe aparecer una nueva fila con la informacion en la tabla de "Productos que estas vendiendo". Despues hacemos logout.
  • PruebaCompleta.feature : El objetivo de esta prueba e sprobar el funcionamiento de la aplicacion probando todas sus principales funciones (usando las pruebas descritas anteriormente). Primero hacemos login con el usuario tipo ADMIN (login-Admin.feature). Despues el admin se encarga de incluir un nuevo producto en la tienda pinchando en el boton de "Nuevo Producto" dentro de la pagina "Administrar Productos". Al pulsar en el boton accedemos a una pagina con un formulario donde rellenaremos los datos obligatorios. Despues de incluir el producto hacemos logout. A continuación llamamos a la prueba de "precio.feature" donde el usuario alvaro09 fijara un precio por el primer producto de la tienda. Despues llamaremos a "compra.feature" donde el usuario danidassler comprará directamente el primer producto de la tienda aceptand el precio incluido por Alvaro09.
  • restaura.feature: El objetivo de esta prueba es añadir dinero al usuario de danidassler para que se pueda ejecutar la prueba de PruebaCompleta.feature sin tener que reiniciar el servidor.

Nuevos detalles de la aplicacion Web

Hay dos formas de vender tu producto en NewChance: Fijar el precio que quieres obtener por tu producto o pulsar el botón de "vender ya"

Al poner un precio a tu producto, indicas que estas dispuesto a vender tu articulo por ese determinado precio.

Para cada producto, nuestra página listará de menor a mayor los precios puestos por los usuarios que desean vender dicho producto.

El usuario puede determinar el numero de dias que el precio se mantendrá en nuestra plataforma antes de que caduque: 1, 3, 7, 14, 31, 62.

El usuario recibirá un mensaje cuando el precio esté a punto de caducar, para que pueda renovarlo o dejarlo ir.

El usuario solo puede fijar un unico precio por cada producto al mismo tiempo.

Además el usuario puede editarlo y/o cancelarlo en todo momento (siempre y cuando no se haya realizado ya la venta).

Si el usuario fija un precio por un producto y este es el más bajo de todos los que existen listados, su precio será denominado: "Precio más bajo".

Si el usuario tiene el "Precio más bajo", los compradores que realicen una puja coincidente con el precio, comprarán el producto automaticamente del usuario. Los compradores realizarán esta acción mediante el botón "Comprar ya".

Si otro usuario lista un precio más bajo que el tuyo, tu precio seguirá activo y en el momento que vuelva a ser el precio mas bajo volverás a la misma situaciónn descrita anteriormente.

Si no quieres esperar, simplemente pulsa el botón "Vender ya", para vender tu producto a la puja más alta ofrecida por un comprador.

En tu cuenta de usuario aparecerá una tabla de ventas con las categorias "Pendientes", "Confirmadas", "Historial".

"Pendientes" mostrará la informacion de los precios que tienes activos en la plataforma.

"Confirmadas" aquellos precios de los articulos que han sido aceptados por compradores y que aún debes enviar.

"Historial" aquellas ventas totalmente completadas.

Hay dos formas de comprar un producto en NewChance: hacer una puja por un producto o pulsar el botón de "Comprar ya"

Hacer una puja refleja tu inteción de comprar un producto por ese determinado precio.

Para cada producto, nuestra página listará de mayor a menor las pujas realizadas por los usuarios que desean comprar dicho producto.

El usuario puede determinar el numero de dias que la puja se mantendrá en nuestra plataforma antes de que caduque: 1, 3, 7, 14, 31, 62.

El usuario recibirá un mensaje cuando la puja esté a punto de caducar, para que pueda renovarla o dejarlo ir.

El usuario solo puede fijar una puja por cada producto al mismo tiempo (esto habria que pensarlo, porque no tiene sentido que haga más de una puja para el mismo producto ya que la úica que "sirve" de verdad es la puja que haga de mayor cantidad).

Además el usuario puede editarla y/o cancelarla en todo momento (siempre y cuando no se haya realizado ya la venta).

Si el usuario fija una puja por un producto y esta es la más alta de todas las que existen listadas, su puja será denominada: "Puja más alta".

Si el usuario tiene la "Puja más alta", los vendedores que fijen un precio coincidente con esta puja, venderán el producto automaticamente al usuario. Los vendedores realizarán esta acción mediante el botón "Vender ya".

Si otro usuario lista una puja más alta que la tuya, tu puja seguirá activa y en el momento que vuelva a ser la puja mas alta volverás a la misma situaciónn descrita anteriormente.

Si no quieres esperar, simplemente pulsa el botón "Comprar ya", para comprar el producto al precio más bajo listado por un vendedor.

En el perfil de usuario aparecerán las mismas tablas descritas en el apartado de vendedores pero en este caso con pujas y compras.

Otros detalles

No es necesario tener credito/dinero en la aplicación para hacer pujas, sin embargo, si una de tus pujas es aceptada, deberás tener el dinero suficiente para completar la trasacción.

Si en el momento en el que se acepta tu puja no posees el saldo suficiente, la transacción no se completará, tu puja será eliminada, tus valoraciones de usuario se verán afectadas

Si es reincidente, se te cobrará una multa y/o el administrador podrá bloquear tu usuario.

Si eres vendedor, deberás marcar unas casillas asegurando que tu pproducto es original y está nuevo a la hora de listar un precio en la app.

Si vendes el producto, también te comprometes a enviar a la empresa el producto en un plazo de 2 dias laborables.

Si estos requisitos no se cumplen, la venta no se efectuará, el dinero será devuelto al vendedor, y el vendedor cubrirá los gastos del envío de vuelta de su producto.

Si es reincidente, se te cobrará una multa y/o el administrador podrá bloquear tu usuario.

Existirá la opcion de que los compradores y vendedores tengan notificaciones/mensajes sobre como evolucionan los precios y pujas de los productos en los que estan involucrados.

Detalles de la aplicación web

¿Tienes problemas a la hora de encontrar un artículo de tu marca favorita que se ha agotado? ¿Cansado de pagar precios que superan el valor de mercado?

New Chance es una página de compra-venta de artículos (ropa, zapatillas, electrónica, relojes…) donde los usuarios se encontrarán con un sistema similar al de una subasta, donde podrán añadir pujas y los vendedores solicitar precios por los productos listados en la página web. La página actúa como intermediario entre el comprador y el vendedor, una vez se realiza una compra, el vendedor tiene un límite de tiempo para enviar el producto a New Chance, una vez en las oficinas de nuestra página, se verificará que el producto es nuevo y original y se envía al comprador.

Para asegurarnos de que no se comete ningún fraude se emitirán condiciones de uso, sancionarán con multas a los vendedores si sus productos no se cumplen las condiciones deseadas o se retrasan y se controlarán los métodos de pago y fondos de los usuarios (sistema "ESCROW").

Nuestra página usará WebSockets para la actualización instantánea de las pujas, las cuales tendrán que actualizarse rápidamente cuando los usuarios pongan nuevas pujas/precios en los productos o estas pujas/precios se acepten por parte de los usuarios y se produzca la venta de los productos.
También sera necesario para mantener los productos de la página actualizados correctamente cuando el usuario administrador incluya nuevos productos o cambie las características de estos.
El uso de WebSockets también nos seria útil para que los usuarios puedan recibir notificaciones instantáneas cuando estos sucesos nombrados se produzcan con algún producto que quieran seguir o estén relacionados con el (tengan una puja o estén esperando un comprador).

Roles de usuario:

  • Usuario no registrado -> Estos usuarios que pueden realizar acciones como :
    • Ver productos y sus precios.
    • Ver evolución del producto en el mercado.
    • Ver datos y características de este y navegar por la página.
  • Usuario registrado -> Estos usuarios, además de poder realizar las acciones de los usuarios no registrados, tienen dos roles: El comprador o el vendedor
    • Comprador -> El usuario que desea comprar un producto puede:
      • Hacer una puja.
      • Ver el estado de sus pujas.
      • Comprar directamente el producto al menor precio solicitado por los vendedores de este.
      • Ver el estado de sus compras.
      • Problemas con un producto comprado (daños, retrasos …).
      • Realizar el seguimiento de un producto (al usuario le llegan notificaciones).
    • Vendedor -> El usuario que vende un producto puede:
      • Vender directamente el producto, aceptando la puja más alta que ofrece un comprador sobre este.
      • Consultar el estado de sus ventas.
      • Hacer una propuesta de precio para un producto.
      • Ver las pujas actuales relacionadas con su producto.
      • Realizar el seguimiento de un producto (al usuario le llegan notificaciones).
  • Administrador -> Se encarga de mantener la página web, tiene todos los permisos para modificar las páginas de la aplicación además de administrar y gestionar la base de datos. Las funcionalidades de este usuario son:
    • Insertar un nuevo producto.
    • Actualizar los datos de un producto existente.
    • Eliminar un producto.
    • Verificar la autenticidad de los productos que llegan a los administradores.
    • Asegurarse de que los artículos vendidos lleguen al destino.
    • Poner sanciones económicas a usuarios.
    • Eliminar usuarios de forma temporal o permanente en función de su comportamiento.
